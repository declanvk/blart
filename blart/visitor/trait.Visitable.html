Expand description

The Visitable trait allows Visitors to traverse the structure of the implementing type and produce some output.

Required Methods

source

fn super_visit_with<V: Visitor<T>>(&self, visitor: &mut V) -> V::Output

This function provides the default traversal behavior for the implementing type.

The implementation should call visit_with(visitor) for all relevant sub-fields of the type. If there are no relevant sub-fields, it should just produce the default output.

fn visit_with<V: Visitor<T>>(&self, visitor: &mut V) -> V::Output

This function will traverse the implementing type and execute any specific logic from the given Visitor.

This function should be override for types that have corresponding hooks in the Visitor trait. For example the Visitable implementation for InnerNode4 looks like:

ⓘ
impl<T> Visitable<T> for InnerNode4<T> {
    ...
  
    fn visit_with<V: Visitor<T>>(&self, visitor: &mut V) -> V::Output {
        visitor.visit_node4(self)
    }
}

The call to visitor.visit_node4(self) allows the visitor to execute specific handling logic.

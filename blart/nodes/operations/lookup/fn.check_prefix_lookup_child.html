Expand description

For the given InnerNode, check the node prefix, then lookup the child based on the search depth.

If the prefix does not match, it returns None. If there is no matching child for the key byte, it returns None.

§Safety

  • No other access or mutation to the inner_ptr Node can happen while this function runs.
